Detailed Description

Middleware for controlling multiple LEDs with flexible blinking patterns.

This module provides an interface for controlling LED indicators with configurable blinking speeds and modes. Features include:

  • Multiple predefined blinking speeds (slow, medium, fast)
  • Independent control for up to 4 LEDs
  • Non-blocking operation using software counters
  • Simple APIs for LED pattern configuration

◆ E_Indicator_Blink_Period_TypeDef

enum E_Indicator_Blink_Period_TypeDef

LED blinking period enumeration.

Defines standard blinking periods as multiples of system sampling time. The actual time period is calculated as: T_blink = period_value * T_sampling

Example with T_sampling = 1ms:

  • SLOW: T = 500ms (1 Hz)
  • MEDIUM: T = 200ms (2.5 Hz)
  • FAST: T = 100ms (5 Hz)
Enumerator
SLOW 

Slow blinking: 500 * T_sampling.

MEDIUM 

Medium blinking: 200 * T_sampling.

FAST 

Fast blinking: 100 * T_sampling.

◆ indicator_process_led_()

void indicator_process_led_ ( S_Indicator_Led_Config_TypeDef * pledConfig)

Processes LED blinking based on configured period.

Parameters
[in,out]pledConfigPointer to LED configuration structure
Note
Must be called at system sampling frequency

This function implements the LED blinking logic:

  1. Increments internal counter
  2. Checks if configured period has elapsed
  3. Toggles LED state when period is reached
  4. Resets counter for next cycle

Timing diagram for LED blinking: 

* LED ON   ┌────┐    ┌────┐    ┌────┐
* LED OFF  └────┘    └────┘    └────┘
*          <-T->     <-T->     <-T->
* Where T = period * system_sampling_time
*
Parameters
[in,out]pledConfigPointer to LED configuration structure
Note
This function must be called at a fixed interval (system sampling time) for accurate timing control
Warning
Ensure the calling frequency is higher than twice the fastest configured blinking period to avoid aliasing effects
