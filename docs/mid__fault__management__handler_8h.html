Soft Protection Bits

Bit definitions for fault status register

The fault status is stored in a 16-bit register where each bit represents a specific fault condition. Bits are divided into:

  • Soft Protection (bits 0-11): Requires software intervention
  • Hard Protection (bits 12-15): Immediate hardware shutdown

void fault_init_ ( void )

Initializes the fault management system.

Performs complete initialization of fault management:

  1. Clears all fault status bits to 0
  2. Resets fault history buffer
  3. Prepares system for fresh fault detection

This function should be called:

  • During system startup
  • After system reset
  • Before enabling any power stage

◆ fault_standby_()

void fault_standby_ ( void )

Places fault management system in standby mode.

Performs complete system reset:

  1. Resets hardware comparator protection
  2. Clears temporary fault status
  3. Clears global fault instance status
  4. Sets fault mode to FAULT_NONE

This function should be called:

  • During system shutdown
  • After fault recovery
  • Before system restart

Updates the fault handling mode based on current fault status.

Determines the appropriate fault handling mode by:

  1. First updating all fault status bits
  2. Checking for latching faults (require manual reset)
  3. Checking for auto-recovery faults
  4. Setting mode to normal if no faults detected

Fault modes (in order of priority):

  1. FAULT_LATCH: Requires manual intervention to reset
  2. FAULT_AUTO_RECOVER: System attempts automatic recovery
  3. FAULT_NONE: Normal operation, no faults detected
Parameters
pFaultManagePointer to fault management instance
pVpsuPointer to PSU output voltage measurement
pIpsuPointer to PSU output current measurement
pVacPointer to AC input voltage measurement
pTpfcPointer to PFC temperature measurement
pTllcPointer to LLC temperature measurement
Note
Fault mode hierarchy ensures most severe faults take precedence
