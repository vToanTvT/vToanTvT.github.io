Detailed Description

Implementation of comprehensive fault management system.

Author
ADMIN

This file implements a robust fault management system with features:

  • Fault detection with configurable thresholds
  • Debounce mechanism to prevent false triggers
  • Fault status tracking and history
  • Multiple protection levels (soft/hard)
  • Auto-recovery and latching fault modes

_Bool fault_debounce_ ( _Bool condition,
uint16_t timeout,
uint16_t * counter )

Implements generic fault condition debouncing.

This function provides a generic debounce mechanism for fault detection:

  1. If fault condition is active, increment counter
  2. If counter exceeds timeout, confirm fault
  3. If condition clears, reset counter This prevents spurious fault triggers from noise or transients.
Parameters
conditionTrue if fault condition is currently active
timeoutNumber of consecutive checks required to confirm fault
counterPointer to the static counter for this fault type
Returns
true Fault condition confirmed (counter exceeded timeout)
false Fault not confirmed or condition cleared
Note
Timeout value determines the required duration of fault condition: actual_time = timeout * sampling_period

Updates the fault handling mode based on current fault status.

Determines the appropriate fault handling mode by:

  1. First updating all fault status bits
  2. Checking for latching faults (require manual reset)
  3. Checking for auto-recovery faults
  4. Setting mode to normal if no faults detected

Fault modes (in order of priority):

  1. FAULT_LATCH: Requires manual intervention to reset
  2. FAULT_AUTO_RECOVER: System attempts automatic recovery
  3. FAULT_NONE: Normal operation, no faults detected
Parameters
pFaultManagePointer to fault management instance
pVpsuPointer to PSU output voltage measurement
pIpsuPointer to PSU output current measurement
pVacPointer to AC input voltage measurement
pTpfcPointer to PFC temperature measurement
pTllcPointer to LLC temperature measurement
Note
Fault mode hierarchy ensures most severe faults take precedence

Updates fault status based on all monitored parameters.

Comprehensive fault detection routine that:

  1. Checks all protection thresholds
  2. Applies debouncing to prevent false triggers
  3. Updates fault status register bits
  4. Handles both LLC and PFC stage protections

Protection types monitored:

  • Over/Under Voltage Protection (OVP/UVP)
  • Over Current Protection (OCP)
  • Short Circuit Protection (SCP)
  • Over Temperature Protection (OTP)

pVacPointer to AC input voltage measurement (Q15)

Note
